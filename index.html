Readme

Main problem witn two-columns layout - equal height columns. There are many workaround to do this, with fixed, fluid, source-ordered columns.

Faux columns and modifications

Floated columns, negative margins, position relative for stacking order.

Background image

Most basic and old-school. Set image on background of container, and use it to make equal-height columns. Cons - http-request for image, maintain, fixed column.

Although, we can use it for fluid-width two-column layout if we make image wide enougth and relatively position it in background.

 
.container {
	width: 980px;
	margin: 0 auto;
	overflow: hidden;
	background: url(img/columns-bg.png) 30% 0 repeat-y; /* like this */
	}

Huge padding/margins

Same as columns with image, but with border as a faux column. Big pagging for equal height, and negative margin to shift content back on its place.

Example 
.container {
	width: 980px;
	margin: 0 auto;
	overflow: hidden; /* we need this to clip huge paddings (and clear floats) */
	}
.sidebar-block {
	width: 300px; /* only pixels in this case, because border size can't be anything else */
	float: left;
	position: relative; /* we need this to bring column to top (positioned elements are higher in stack) */
	margin-right: -300px; 
	}
.content-block {
	width: 680px;
	background-color: blue;
	float: left;
	padding-bottom: 32111px; /* we set this to make column height */
	margin-bottom: -32111px; /* and shift back content */
	border-left: 300px solid red; /* this is other column background */
	}

CSS3 Gragients

For modern browsers faux columns can be setted with gradient and apropriate color stops.

Pseudo elements

Column backgrounds are made with empty 100%-height pseudo-elments, absolute positioned under apropriate columns.

Example 
.container {
	width: 980px;
	margin: 0 auto;
	position: relative;
	z-index: 1;
	overflow: hidden;
	background-color: #555;
	}
.sidebar-block {
	width: 30%;
	position: relative;
	float: left;
	}
.content-block {
	width: 70%;
	float: left;
	}

/* And here is our columns backgrounds */
.container:before, .container:after {
	content: " ";
	position: absolute; /* get them out of the document flow */
	z-index: -1; /* position them in stack below anything else */
	top: 0;
	height: 100%;
	}
.container:before {
	width: 30%; /* same as column width */
	background: red;
	}
.container:after {
	left: 30%;
	width: 70%;
	background-color: blue;
	}

Table cells

We can imitate good old table with css `table-cell` property. But without source-ordering abilities in this case.

MJT

Matthew James Taylor method. Floated container have height as its floated children. With aditional divs we shift columns to make columns backgrounds, and shift content back into place.

Example 
  
.container-outer {
	width: 980px;
	margin: 0 auto;
	background-color: red; /* content-block background */
	overflow: hidden;
	}
.container-inner {
	float: left;
	background-color: #555; /* sidebar background */
	position: relative;
	right: 75%; /* we shift this to reveal content background */
	}
.sidebar-block {
	width: 25%;
	float: left;
	position: relative;
	left:75% /* we shift this to bring content back */
	}
.content-block {
	width: 680px;
	float: left;
	position: relative;
	left:75% /* we shift this to bring content back */
	}

Flexboxes

In brave new world we can use flexboxes.

Example 
  
.container {
	display: flex; /* set context to flexbox */
	/* vendor prefixes for each browser */
}
.sidebar-block {
	background-color: red;
	width: 30%;
}
.content-block {
	background-color: blue;
	width: 70%;
}
